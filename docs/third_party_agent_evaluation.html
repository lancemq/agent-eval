第三方 Agent 评测机制说明

AgentEval 将任意第三方 Agent 包装成统一接口，通过插件生成任务、调用 Agent、使用评判器评分，并输出 JSON / HTML / Markdown 报告。

整体流程

1第三方 Agent 包装为 AgentUnderTest / CallableAgent
2EvaluationOrchestrator 初始化插件与运行上下文
3插件通过 generate_tasks() 生成评测任务
4插件通过 execute_task() 调用第三方 Agent
5插件通过 evaluate() 使用 Judge / Scorer 评分
6编排器聚合 summary、plugin_results、task_results
7生成 JSON / HTML / Markdown 报告

1. 第三方 Agent 接入方式

AgentEval 不要求第三方 Agent 使用特定框架，只要求能被包装成统一接口。

核心接口位于：agent_eval/orchestrator/agent.py:7

必须实现

generate(prompt: str) -> str
chat(messages: list[dict]) -> str

工具调用评测可选实现

act(state: dict, available_tools: list, goal: str) -> dict

act() 返回动作示例：

{
  "type": "tool_call",
  "tool": "calculator",
  "params": {"expression": "15 * 23 + 42 / 6"}
}
若第三方 Agent 未实现 act()，框架会使用默认实现，将状态、工具、目标拼成 prompt，再尝试解析 Agent 返回的 JSON。

8. 自定义插件

用户可以实现自己的 BasePlugin，评测特定业务场景。

@register_plugin
class MyPlugin(BasePlugin):
    name = "my_plugin"
    evaluation_type = EvaluationType.CUSTOM
    supported_dimensions = ["creativity"]

    def setup(self, config):
        self.test_cases = config.get("test_cases", [])

    def generate_tasks(self, context):
        return self.test_cases

    def execute_task(self, task, context):
        return context.agent_under_test.generate(task["instruction"])

    def evaluate(self, task, output, context):
        score = 1.0 if len(output) > 10 else 0.0
        return EvalResult(
            plugin_name=self.name,
            evaluation_type=self.evaluation_type,
            score=score,
            raw_score={"length": len(output)},
            details={"instruction": task["instruction"]},
            artifacts=[output],
            passed=score >= 0.5,
            execution_time_ms=0,
            task_id=task.get("id", ""),
        )

10. 当前限制与后续优化方向

限制说明优化方向
插件间串行插件内部任务并发，但插件之间顺序执行支持 parallel_plugins
overall_score 是插件均分小插件与大插件权重相同支持 macro / micro average
LLM 调用缺少 retry / timeout429 / 5xx 时无统一重试增加 backoff + rate limit
Sandbox 能力有限复杂工具链环境模拟不足增强 Docker sandbox
维度分数粒度粗维度分目前由插件平均分映射支持 task-level 维度分
Judge 异常记为 0 分评判器故障与 Agent 表现差混淆区分 judge_error
第三方 Agent 与插件都是本地 Python 代码加载，评测不可信代码时需要使用隔离环境。
